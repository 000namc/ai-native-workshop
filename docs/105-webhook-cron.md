# 105. Webhook & Cron — 자동화 트리거

> Session 1: AI 활용

## Webhook — 이벤트가 발생하면 자동 실행

### 이것은 무엇인가요?

Webhook은 특정 이벤트가 발생했을 때 자동으로 작업을 실행하는 트리거입니다.

### 예시

- **PR 머지 → 릴리즈 노트 초안 자동 작성**: 개발팀이 코드를 머지하면, AI가 변경 내용을 읽고 릴리즈 노트를 만들어줍니다
- **이벤트 발생 → 이메일 초안 작성**: 특정 조건이 충족되면 자동으로 이메일 초안을 작성합니다

## Cron — 정해진 시간에 반복 실행

### 이것은 무엇인가요?

Cron은 정해진 시간에 반복적으로 작업을 실행하는 스케줄러입니다.

### 예시

- **매일 아침 → 어제 Slack/Jira 요약 정리**: 출근하면 어제의 대화와 이슈가 정리되어 있습니다
- **매주 금요일 → 주간 보고 초안 생성**: 한 주간의 활동을 자동으로 요약해줍니다

## 핵심 포인트

- Webhook = "무슨 일이 생기면" 자동 실행
- Cron = "정해진 시간에" 반복 실행
- 반복적인 정리/보고 업무를 AI에게 맡길 수 있다
