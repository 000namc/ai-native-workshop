# 07. Git 기본 — 버전관리 개념 이해하기

> Session 2: 개발 환경 기본

## 이것은 무엇인가요?

Git은 파일의 변경 이력을 관리하는 도구입니다.
"최종.pptx", "최종_진짜최종.pptx" 대신, 모든 변경 기록을 체계적으로 관리합니다.

## 왜 알아야 하나요?

- 코딩 에이전트가 만든 코드를 안전하게 관리할 수 있습니다
- 문제가 생겼을 때 이전 상태로 되돌릴 수 있습니다
- 여러 사람이 동시에 작업할 수 있습니다

## 핵심 개념

### Commit — 스냅샷 저장

파일의 현재 상태를 사진 찍듯 저장하는 것입니다.

```
commit 1: "로그인 페이지 추가"
commit 2: "비밀번호 찾기 기능 추가"
commit 3: "디자인 수정"
  └── 언제든 commit 1 시점으로 돌아갈 수 있습니다
```

### Branch — 갈래 나누기

원본을 건드리지 않고 새로운 시도를 할 수 있는 평행 세계입니다.

```
main ──────────────────────────
         \
          feature/new-button ── 여기서 자유롭게 실험
```

### Repository — 프로젝트 저장소

- **로컬 저장소**: 내 컴퓨터에 있는 프로젝트
- **원격 저장소(GitHub)**: 클라우드에 올린 프로젝트

### 기본 흐름

```
파일 수정 → git add (준비) → git commit (저장) → git push (공유)
```

| 명령어 | 하는 일 | 비유 |
|--------|---------|------|
| `git init` | 저장소 만들기 | 새 프로젝트 시작 |
| `git add` | 변경 사항 준비 | 택배 상자에 물건 넣기 |
| `git commit` | 변경 사항 저장 | 택배 상자 봉인 |
| `git push` | 원격에 업로드 | 택배 발송 |
| `git pull` | 원격에서 가져오기 | 택배 수령 |

## 정리

- Git = 파일 변경 이력 관리 도구
