# 302. 작업환경 설계 — 에이전트가 일하기 편한 환경

> Session 3: 코딩 에이전트 기본

## 이것은 무엇인가요?

에이전트가 효율적으로 일할 수 있도록 프로젝트 환경을 설계하는 방법입니다.

## 왜 중요한가요?

- 잘 정리된 환경 = 에이전트의 실수가 줄어듭니다
- 명확한 구조 = 원하는 결과를 더 잘 얻을 수 있습니다

## 핵심 원칙

### 1. 폴더 구조를 명확하게

```
my-project/
├── CLAUDE.md          ← 에이전트가 가장 먼저 읽는 파일
├── src/               ← 소스 코드
├── build/             ← Docker 설정
├── docs/              ← 문서
└── package.json       ← 프로젝트 설정
```

### 2. 요구사항을 구체적으로

```
❌ "타이머 앱 만들어줘"
✅ "25분 집중 / 5분 휴식 뽀모도로 타이머. 세션 완료 시 DB에 저장. 오늘의 기록 목록 표시."
```

### 3. 작업 단위를 나누어서

```
❌ "전체 앱을 한 번에 만들어줘"
✅ "먼저 타이머 UI를 만들어줘" → "이제 DB 연동을 추가해줘" → "Docker 설정을 만들어줘"
```
