# 303. 에이전트 + 도구 — Docker, Git 연동 활용

> Session 3: 코딩 에이전트 기본

## 이것은 무엇인가요?

에이전트에게 Docker와 Git이라는 도구를 쥐어주면, 단순히 코드를 작성하는 것을 넘어 환경 구성과 버전 관리까지 스스로 수행합니다.

## Docker 연동

### 에이전트가 할 수 있는 일

- Dockerfile, docker-compose.yml 작성
- 컨테이너 빌드 및 실행
- 실행 결과를 확인하고 오류 수정

### 작업 흐름

```
사용자: "뽀모도로 타이머를 Docker로 실행 가능하게 만들어줘"

에이전트가 하는 일:
1. 코드 작성
2. Dockerfile 작성
3. docker-compose.yml 작성
4. docker compose up으로 실행
5. 오류가 있으면 수정 후 재실행
```

## Git 연동

### 에이전트가 할 수 있는 일

- 변경사항을 commit으로 저장
- 적절한 commit 메시지 작성
- GitHub에 push

### 작업 흐름

```
사용자: "지금까지 작업한 내용을 커밋해줘"

에이전트가 하는 일:
1. git status로 변경사항 확인
2. git add로 파일 준비
3. git commit으로 저장 (의미있는 메시지와 함께)
```

## 정리

- 에이전트 + Docker = 코드 작성부터 실행 환경까지 한 번에
- 에이전트 + Git = 버전 관리까지 자동화
- 도구를 연결하면 에이전트의 활용도가 크게 높아집니다
